# Equation and Algorithm Teaching

Use this when the paper contains equations, objectives, losses, update rules, algorithms, or pseudocode.

## Equation Translation Pattern

For each important equation:

1. Name what the equation is doing before explaining symbols.
2. List variables in human language.
3. Explain the left side as the goal or quantity being computed.
4. Explain the right side as the recipe.
5. Describe what changes if one term becomes larger or smaller.
6. Connect the equation to the method pipeline or experiment.

## HTML Teaching Shape

Use a translation block:

- Left side: the exact equation or short pseudocode excerpt when allowed, or a faithful compact restatement.
- Right side: line-by-line explanation.

Add a variable card grid after the equation when there are 3+ symbols.

## Do Not

- Say "simply" before a hard idea.
- Explain only notation while skipping purpose.
- Treat all equations as equally important.
- Hide uncertainty when the paper's notation is ambiguous.

## Good Quiz Styles

- "If this regularization term increases, what behavior should you expect?"
- "Which part of the objective encourages X?"
- "A reproduction gets unstable training; which variable would you inspect first?"
- "Which assumption breaks if the input distribution changes?"
