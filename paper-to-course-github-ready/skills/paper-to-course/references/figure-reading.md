# Figure and Table Reading

Use this when teaching figures, tables, diagrams, plots, or screenshots from a paper.

## Figure Walkthrough

For each central figure:

1. State the question the figure answers.
2. Identify axes, panels, colors, and symbols.
3. Explain the expected pattern if the paper's claim is true.
4. Explain what the figure actually shows.
5. Name one caveat or alternative reading.

## Visual Course Elements

Use:
- Pattern cards for multi-panel figures
- Flow diagrams for architecture figures
- Step cards for algorithms
- Scenario quizzes for interpreting tables
- Callouts for "what this figure proves" and "what it does not prove"

## Evidence Map

Use this structure when a paper has many results:

```text
Claim A -> Figure 2 + Table 1 -> strong/medium/weak
Claim B -> Ablation in Table 3 -> strong/medium/weak
Claim C -> Qualitative examples -> suggestive only
```

## Avoid

- Describing visual appearance without explaining evidence.
- Saying a figure "clearly shows" when the claim depends on hidden setup.
- Ignoring error bars, sample size, benchmark choice, or variance.
